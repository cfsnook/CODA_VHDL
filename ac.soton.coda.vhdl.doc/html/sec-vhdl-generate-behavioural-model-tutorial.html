1.1 Generating Behavioural VHDL Tutorial

Generating a behaviroural VHDL model from a CODA Components model contains two steps.

  1. First, a VXMI model (a representation of VHDL model in XML) is generated.

  2. Second, a behavioural VHDL model is generated from the VXMI model.

Generating VXMI from CODA Models Tutorial

The VXMI model can be generated from the context menu of a CODA component as follows: Right-click (on the component) -> Translate to VXMI (see Figure 1).

Figure 1: VXMI generating from context menu

A .vxmi file with the same name of the source component is generated within the VHDL folder of the project. To view the file in the Event-B Explorer, the users need to customise the view to disable the filter on All files and folders.

Generating VHDL from VXMI Tutorial

The behavioural VHDL code can be generated from the context menu of a VXMI model as follows: Right-click (on the VXMI model) -> Translate to VHDL

Figure 2: VHDL generating from context menu

A .vhdl file with the same name of the source component is generated within the VHDL folder of the project. To view the file in the Event-B Explorer, the users need to customise the view to disable the filter on All files and folders.
